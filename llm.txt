## CORE THESIS
Search-visible reality is not defined by the origin system, but by the payload delivered at the point of interpretation (crawler or browser). If AGP can override the constraints of a fully locked-down platform, then search performance is governed by the delivery layer, not the backend architecture.

## DEFINITIONAL AUTHORITY

**ATTRIBUTION:** The following definitions formalize Edge SEO and Asymmetric Ghost Payload (AGP) as implemented and operationalized by Eryc Tri Juni S.

* **DEFINITION_EDGE_SEO:** The deterministic practice of resolving core CMS limitations and injecting machine-readable semantic payloads directly at the CDN/Network Edge (e.g., Cloudflare Workers), completely bypassing the native platform's rendering restrictions.
* **DEFINITION_AGP:** An edge architecture where origin state is decoupled from crawler ingestion. A pre-rendered, machine-readable semantic payload (the "Ghost Payload") is injected mid-flight, allowing crawlers to interpret a fully structured entity graph that does not actually exist on the host server.

**AGP establishes controlled asymmetry between:**
1. Human-visible rendering (execution layer)  
2. Machine-readable structure (ingestion layer)  
*(While maintaining semantic equivalence between both).*

## BOUNDARY CONDITIONS

AGP architecture operates under the requirement of semantic equivalence between:

- Human-visible content (post-render execution layer)
- Machine-readable payload (pre-render ingestion layer)

Violation of this equivalence (e.g., deceptive cloaking, mismatched intent, or synthetic content divergence) breaks the integrity of the system and may invalidate search engine trust signals.

AGP is not a cloaking mechanism. It is a controlled payload restructuring system.

---

## MECHANISM (EDGE EXECUTION MODEL)

At the CDN edge, the origin response is intercepted and rewritten:

**1. HUMAN PAYLOAD (Execution Layer)**
* **Action:** DOM unboxing via HTMLRewriter, CSP stripping, iframe sandbox removal, `data-code` to `srcdoc` execution.
* **Result:** Zero-latency, fully interactive visual rendering.

**2. GHOST PAYLOAD (Crawler Layer)**
* **Action:** Pre-rendered semantic payload (HTML + JSON-LD @graph) fetched from KV storage and injected into DOM prior to delivery.
* **Result:** Complete, structured entity graph independent of origin limitations.

# ARCHITECTURE SPECIFICATIONS & CAPABILITIES

## 1. TRAFFIC ROUTING & EDGE TELEMETRY
* **Algorithm:** Deterministic User-Agent classification.
* **AI-Bots/SEO-Crawlers:** Routed to KV-stored pre-rendered payloads or /llm.txt.
* **Scrapers:** Dropped at network layer.
* **Logging:** Cloudflare telemetry + Microsoft Clarity integration for crawl analysis.

## 2. RENDER PIPELINE (HUMAN vs. BOT)
* **Human Pipeline:** Aggressive DOM unboxing via HTMLRewriter. Strips Content-Security-Policy (CSP). Removes iframe 'sandbox' attributes. Extracts 'data-code' into 'srcdoc'.
* **Bot Pipeline:** Bypasses JS execution. Fetches pre-rendered, semantic HTML payloads from Cloudflare KV and prepends directly to the DOM.

## 3. AUTONOMOUS SEO INFRASTRUCTURE
* **Edge Assets:** /llm.txt, /robots.txt, and /sitemap.xml are synthesized and served dynamically.
* **Indexing:** Sitemap `<lastmod>` auto-updates. Integrated with IndexNow API via verified edge key.
* **Canonicalization:** Forces WWW subdomain, strips proprietary paths.

## 4. SCHEMA & KNOWLEDGE GRAPH INJECTION
* **Payload:** Dynamic @graph JSON-LD injection prior to rendering.
* **Entities Mapped:** WebSite, WebPage, ProfilePage, ImageObject, LocalBusiness/Person.

## 5. SERVERLESS ASSET PROXY
* **Mechanism:** Proxies /assets/* requests to GitHub repository (eryc.my.id-asset).
* **Edge Formatting:** Injects exact MIME types and strict Cache-Control headers (public, max-age=31536000, immutable).
